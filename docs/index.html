HTML basics

You know R and/or Python! HTML is not so different. Instead of scientific computing, however, HTML encodes how to display information.

1
Core concept

Tags are like function calls

Suppose you wanted to make text bold. In R, you would wrap your content in a function that looks something like this: bold("hello"). In HTML you wrap content in a tag: <strong>hello</strong> (strong means bold here). The opening tag turns something on, the closing tag (with the /) turns it off.

# R — you wrap content in functions
bold("hello")
italic("world")

<!-- HTML — you wrap content in tags -->
<strong>hello</strong>        <!-- bold -->
<em>world</em>             <!-- italic -->
<p>a paragraph</p>       <!-- paragraph -->
<h1>big heading</h1>     <!-- heading -->
<h2>smaller</h2>         <!-- subheading -->
2
Core concept

Nesting is like nested lists in R

Tags can go inside other tags, just like nested lists in R. The structure is a tree, where outer tags are parents and inner tags are children. Indentation makes it readable but doesn't affect how it works (unlike Python, where indentation is necessary).

# R nested list
list(
  title = "My project",
  authors = list("Alice", "Bob")
)

<!-- HTML nesting -->
<section>
  <h2>My project</h2>
  <ul>
    <li>Alice</li>
    <li>Bob</li>
  </ul>
</section>
3
Core concept

Attributes are like named arguments

Some tags need extra information like where a link goes, or the name of the image file to display You pass this as an attribute inside the opening tag, just like named arguments in a function call.

# R named arguments
read.csv(file="data.csv", header=TRUE)

<!-- HTML attributes -->
<a href="https://github.com">My repo</a>  <!-- link -->
<img src="figures/km.svg" alt="KM curve"> <!-- image -->

<!-- class= lets you apply CSS styles -->
<p class="big-text">This paragraph is styled.</p>
4
Core concept

HTML is the content, and CSS governs the global formatting

If you want to change what the page says, edit the HTML. If you want to change how it looks, edit the CSS.

# R: data and presentation are separate
df |> ggplot(aes(x, y)) + geom_line() + theme_minimal()

<!-- HTML: content -->
<p class="intro">We analyzed 4,449 patients.</p>

/* CSS: presentation — in style.css */
.intro {
  font-size: 1.2rem;
  color: #0f4c81;
  font-style: italic;
}
5
Reference

The tags we'll use

Here is a set of useful tags for your project page:

<!-- HEADINGS -->
<h1>Page title</h1>
<h2>Section heading</h2>
<h3>Subsection</h3>

<!-- TEXT -->
<p>A paragraph of text.</p>
<strong>Bold text</strong>
<em>Italic text</em>

<!-- LISTS -->
<ul>                       <!-- bullet list -->
  <li>First item</li>
  <li>Second item</li>
</ul>

<!-- LINKS AND IMAGES -->
<a href="https://...">Link text</a>
<img src="figures/km.svg" alt="KM curve">

<!-- LAYOUT CONTAINERS -->
<section> ... </section>  <!-- a page section -->
<div> ... </div>          <!-- a generic box -->

<!-- FIGURE WITH CAPTION -->
<figure>
  <img src="figures/km.svg" alt="KM curve">
  <figcaption>Figure 1. KM curves by BRCA2 status.</figcaption>
</figure>
6
Customisation

Changing colors and fonts

Open style.css. The very top has a block called :root { } — this is the only section you need to edit to completely restyle the page. Change a hex code, save, and your browser preview updates instantly.

/* style.css — edit ONLY this block to restyle */
:root {
  --ink:     #1a1a2e;   /* headings                */
  --body:    #3d3d5c;   /* body text               */
  --muted:   #7c7c9a;   /* captions and labels     */
  --accent:  #0f4c81;   /* links and highlights    */
  --surface: #f4f6fb;   /* section background      */
  --border:  #e2e4ed;   /* lines and dividers      */

  --font-main: Georgia, serif; /* swap for any Google Font */
}

/* To use a Google Font:
   1. Go to fonts.google.com
   2. Pick a font → "Get font" → copy the <link> tag
   3. Paste it into the <head> of index.html
   4. Then change --font-main here:              */
  --font-main: 'Lora', serif;
Find hex codes: coolors.co generates palettes (hit spacebar to shuffle) · colorhunt.co has curated four-color sets · colororacle.org shows what your page looks like to people with common color vision impairments.

2
Concept

Page structure — the HTML skeleton

For our project, you need five sections: a hero, a data description, a methods section, a results section, and author info. Use semantic HTML — <section>, <header>, <footer> — not just <div>s.

<!-- docs/index.html -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My project with MSK-CHORD</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <header> ...nav... </header>
  <main>
    <section id="hero">    <!-- title, pitch, links -->
    <section id="data">    <!-- MSK-CHORD cohort -->
    <section id="methods"> <!-- model, covariates -->
    <section id="results"> <!-- KM curves, HRs -->
    <section id="team">    <!-- authors -->
  </main>
  <footer> ...data access note... </footer>
</body>
3
CSS

Typography and color

Use CSS custom properties for your palette. Choose clean, easy-to-read fonts. Use max-width: 100ch on paragraphs for readable line length.

/* style.css */
:root {
  --ink:    #1a1a2e;   /* headings       */
  --body:   #3d3d5c;   /* body text      */
  --muted:  #7c7c9a;   /* captions, meta */
  --accent: #0f4c81;   /* MSK blue       */

  --font-serif: Georgia, serif;
  --font-sans:  system-ui, sans-serif;
  --font-mono:  monospace;
}

h1, h2, h3 {
  font-family: var(--font-serif);
  color: var(--ink);
}
h1 { font-size: clamp(2rem, 5vw, 3.2rem); }

p  { max-width: 90ch; }   /* ~90 chars/line */

code {
  font-family: var(--font-mono);
  background: #f4f6fb; padding: 1px 5px;
}

HTML + CSS · 10 min

The hero section

What is a hero section?

  • A hero section is the big block at the top of a webpage that gives the main goals of the page etc.
  • Your hero should answer in 3 seconds:
    • What dataset?
    • What question?
    • What did you find?
  • A common structure is: "We [method] to [outcome] using [data], finding [result]." But please feel free to use your own creativity and style to make this personal (and help it stand out!).
username.github.io/msk-chord-survival
MSK-CHORD · 2026
Genomic Predictors of Survival
in the MSK-CHORD Cohort

HTML + CSS · 10 min

Results section — adding your figures

First export your figure from R into the docs/figures/ folder. Then add a <figure> block to your HTML pointing to it. (Or, use a figure you've already generated for your report, and just move it to the docs/figures/ folder!)

# Step 1: export from R into your docs/figures/ folder
ggsave("docs/figures/figure1.png", my_plot, width=6, height=4, dpi=144)
<!-- Step 2: add this to your index.html -->
<figure>
  <p class="fig-finding">Your key finding in one sentence.</p>
  <img src="figures/figure1.png" alt="description of figure">
  <figcaption>Figure 1. Your caption here.</figcaption>
</figure>
Two figures? Just copy the whole <figure> block and change the filename and caption. They'll sit side by side automatically on a wide screen and stack on a narrow one.
6
Deploy

Deploy and share

Almost done! Add a data access note to your footer, then push your docs/ folder to Github and enable Pages in Settings.

  • ✓ docs/index.html created
  • ✓ KM curves and forest plot exported as SVG
  • ✓ Analysis code committed to repo
  • Data access note added to footer
  • Settings → Pages → Branch: main / Folder: /docs
<!-- Add this to your index.html -->
<footer>
  <p>
    Patient-level MSK-CHORD data are available through
    <a href="https://cbioportal.org">cBioPortal</a>
    under controlled access. Analysis code is available
    under the MIT license.
  </p>
</footer>

Now let's create a GitHub repository to hold your project.

First, create a repo: Go to github.com → click the + in the top right → New repository → give it a name like msk-chord-survival → set it to Public → click Create repository.
Upload by hand: Go to your repo on github.com → click Add file → Upload files → drag your whole docs/ folder in → click Commit changes. Then go to Settings → Pages → Branch: main / Folder: /docs → Save. Your site will be live within about 60 seconds.
